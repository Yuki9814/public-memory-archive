# 审核清单

## 发布前

- [ ] `neutral_title` 存在，且与事实流程相符。
- [ ] `title` 与 `neutral_title` 不含煽动、嘲讽、动员式词汇。
- [ ] `summary` 存在，避免结论先行。
- [ ] 至少有一个来源。
- [ ] 至少一条时间线绑定 `source_id`。
- [ ] 所有来源都有 `reliability_level`。
- [ ] 关键主张有关联证据，或标为 `UNVERIFIED` / `INSUFFICIENT_EVIDENCE`。
- [ ] D 级材料没有单独支撑结论。
- [ ] `HIGH` privacy actor 有 `privacy_note`。
- [ ] 涉及未成年人时使用 `HIGH` 或 `MINOR_PROTECTED`。
- [ ] 页面文本不包含明显手机号、身份证、住址等敏感信息。
- [ ] B站、小红书、微博外链有中性 description，不是裸链接。
- [ ] 已生成 `event_versions` 快照。
- [ ] 纠错入口开启。
- [ ] 举报入口开启。

## 纠错处理

- [ ] 确认纠错指向：事件、来源、时间线、主张、证据或平台外链。
- [ ] 核对纠错材料来源等级。
- [ ] 接受纠错时生成新版本快照。
- [ ] 拒绝纠错时写明内部原因，避免公开羞辱提交者。

## 举报处理

- [ ] 隐私泄露、人肉、未成年人信息优先级最高。
- [ ] 必要时先下架或隐藏相关字段，再做完整复核。
- [ ] 处理完成后记录 `resolution_notes`。
- [ ] 若为平台外链风险，优先隐藏描述中的可识别信息，而非删除整个事件流程。

## 复核口径

- [ ] 不对性别群体作归因。
- [ ] 不用评论区情绪替代证据。
- [ ] 不把“当事人曾说”写成“事实已经证明”。
- [ ] 不把“平台介入”写成“官方定性”。
- [ ] 不让截图脱离来源页面、发布时间和上下文。
